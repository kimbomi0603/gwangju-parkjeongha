# 검색 노출 (네이버·다음·구글) + 업로드 주의사항

## 1) 이번에 추가된 검색 노출(SEO) 설정
- 모든 페이지에 canonical / Open Graph / Twitter Card / 키워드 / theme-color 추가
- 홈에 Person 구조화 데이터(JSON-LD) — 검색 결과에 후보 정보가 풍부하게 노출되도록
- robots.txt (검색엔진 크롤러 전체 허용) + sitemap.xml (8개 페이지 등록)
- 네이버·구글 사이트 인증용 메타 태그 자리(빈 값) 준비

## 2) 검색 콘솔에 등록하기 (필수)
검색에 잘 잡히려면 한 번만 등록해 주세요.

### 네이버
1. https://searchadvisor.naver.com 접속 → 네이버 로그인
2. "웹마스터 도구" → 사이트 추가 → `https://광주박정하.kr`
3. 사이트 소유확인 방법에서 **HTML 태그** 선택 → 보여주는 `content="xxxxx..."` 값 복사
4. 모든 HTML 페이지(index.html, profile.html 등 8개)의 `<meta name="naver-site-verification" content="">` 의 빈 따옴표 안에 그 값을 붙여넣기 → GitHub 업로드
5. 네이버에서 "소유확인" 버튼 → 통과
6. 좌측 "요청 → 사이트맵 제출" → `sitemap.xml` 입력 → 제출

### 구글
1. https://search.google.com/search-console 접속 → Google 로그인
2. 속성 추가 → URL 접두어 → `https://광주박정하.kr` 입력
3. 소유확인 → HTML 태그 → `content="xxxxx..."` 복사
4. 모든 페이지의 `<meta name="google-site-verification" content="">` 의 빈 따옴표 안에 붙여넣기 → 업로드
5. 구글에서 확인
6. 좌측 "Sitemaps" → `sitemap.xml` 추가 → 제출

### 다음(카카오)
다음은 별도 웹마스터 도구가 사실상 닫혀 있어 자동 크롤로 색인됩니다. 위 네이버/구글 등록을 마치고 robots.txt + sitemap.xml 이 잘 노출되어 있으면 다음도 점차 색인됩니다. 시간이 걸릴 수 있어요.

> 인증값은 매번 새로 발급되니, 위 작업이 부담되시면 캡처해서 보내주세요. 제가 메타 태그 8개 페이지에 한꺼번에 넣어드릴게요.

## 3) 이번 업로드 시 꼭 확인할 점 ⚠️
지난번 업로드에서 **`assets/img/field-01.jpg ~ field-10.jpg` 10장이 빠졌습니다.** (현장 속으로 슬라이드 이미지)
- 이번 ZIP에는 다 들어 있으니, **`assets/img` 폴더를 통째로** 끌어 올려 주세요.
- 새로 추가된 파일들도 잊지 마세요:
  - `robots.txt`, `sitemap.xml` (검색용)
  - `cloudflare-worker.js`, 안내문 md 파일들 (필요 시)

## 4) 후보 소개 사진(고화질)
채팅으로 보내주신 새 고화질 사진은 파일로는 안 들어왔어요(인라인 붙여넣기로 들어옴). 그래서 이번 ZIP의 `assets/img/portrait_main.png`는 이전 버전(480×480)입니다.
- 더 큰 원본을 ZIP으로 한 번 더 보내주시면 교체해 드립니다, **또는**
- 직접 깃허브에서 `assets/img/portrait_main.png` 를 새 고화질 파일로 덮어쓰기 하셔도 됩니다.

## 5) HTTPS 인증서
`https://광주박정하.kr` 아직 인증서가 발급 안 됐습니다.
- GitHub 저장소 → Settings → Pages → "Custom domain" 칸 비우고 저장 → 1분 뒤 `광주박정하.kr` 다시 입력 → 저장. 인증서가 발급되면 (체크표시) "Enforce HTTPS" 체크.
